# Workshop: Messaging API — รู้จักข้อมูลจำเพาะและข้อความทุกประเภทก่อนลงมือส่ง

> ก่อนจะเขียนโค้ดยิงข้อความหาผู้ใช้สักคน คุณควรรู้ก่อนว่า LINE Platform "ยิง URL ที่ไหน" "ยิงได้บ่อยแค่ไหน" และ "ส่งข้อความได้กี่แบบ" — เพราะถ้าเลือกโดเมนผิด ยิงเกิน rate limit หรือใช้ type ผิด API จะตอบกลับ 404/429/400 ทันที บทนี้คือ "แผนที่ใหญ่" ของ Messaging API ที่คุณต้องเปิดไว้ข้างจอ

## ทำไมต้องรู้เรื่องนี้?

ลองนึกภาพว่า Messaging API คือ "ไปรษณีย์ LINE" ที่มีหลายสาขา — สาขาหนึ่งไว้รับ **พัสดุธรรมดา** (ข้อความ text, sticker, flex) อีกสาขาไว้รับ **พัสดุขนาดใหญ่** (ไฟล์รูป ไฟล์วิดีโอ ริชเมนู) ถ้าคุณเอาพัสดุใหญ่ไปส่งที่สาขาพัสดุธรรมดา เขาจะตีกลับทันที

นอกจากสาขาแล้ว แต่ละสาขายังมี "คิว" ของตัวเอง — ยิง broadcast ได้ 60 ครั้ง/ชั่วโมง, ยิง reply/push ได้ 2,000 ครั้ง/วินาที — ถ้าเกินก็โดน `429 Too Many Requests`

และสุดท้าย พัสดุในกล่องก็มีหลายแบบ — text, sticker, image, video, audio, location, imagemap, template, flex, quick reply — แต่ละแบบมีโครงสร้าง JSON ที่ต่างกัน บทนี้จะพาคุณรู้จักทั้งหมดก่อนลงมือเขียนโค้ดจริงในบทถัดไป

## Common specifications

ข้อกำหนดทั่วไปสำหรับ LINE Messaging API รวมถึงชื่อโดเมนของ endpoint, การตอบสนองเมื่อคำขอสำเร็จหรือล้มเหลว, และข้อจำกัดของการใช้งาน (rate limit) ซึ่งมีรายละเอียดดังนี้

### Domain — เลือกโดเมนให้ถูกก่อน ไม่งั้นได้ 404

ใน Messaging API ชื่อโดเมนจะต่างกันไปตามแต่ละ endpoint ดังนั้นควรระวังในการใช้ชื่อโดเมนที่ถูกต้องสำหรับแต่ละ endpoint — คิดง่าย ๆ ว่า `api-data.line.me` ใช้สำหรับ **"data ขนาดใหญ่"** (ไฟล์ binary) ส่วน `api.line.me` ใช้สำหรับ **"ข้อความและ metadata"**

| Domain name	                | Endpoint                  |
| -                             | -                         |
| api-data.line.me              | [Get content](https://developers.line.biz/en/reference/messaging-api/#get-content) 
|                               | [Create audience for uploading user IDs (by file)](https://developers.line.biz/en/reference/messaging-api/#create-upload-audience-group-by-file)
|                               | [Add user IDs or Identifiers for Advertisers (IFAs) to an audience for uploading user IDs (by file)](https://developers.line.biz/en/reference/messaging-api/#update-upload-audience-group-by-file)
|                               | [Upload rich menu image](https://developers.line.biz/en/reference/messaging-api/#upload-rich-menu-image)
|                               | [Download rich menu image](https://developers.line.biz/en/reference/messaging-api/#download-rich-menu-image) |
| api.line.me                   | Other API endpoints  


ตัวอย่าง endpoint ของ **api.line.me** (ส่งข้อความทุกประเภท):
````
POST https://api.line.me/v2/bot/message/reply
POST https://api.line.me/v2/bot/message/push
POST https://api.line.me/v2/bot/message/multicast
POST https://api.line.me/v2/bot/message/narrowcast
GET  https://api.line.me/v2/bot/message/progress/narrowcast
POST https://api.line.me/v2/bot/message/broadcast
````

ตัวอย่าง endpoint ของ **api-data.line.me** (ดึงไฟล์ binary):
````
GET https://api-data.line.me/v2/bot/message/{messageId}/content
GET https://api-data.line.me/v2/bot/message/{messageId}/content/transcoding
GET https://api-data.line.me/v2/bot/message/{messageId}/content/preview
````


### Rate Limit — ยิงได้กี่ครั้งต่อหน่วยเวลา

Messaging API ใช้ข้อจำกัดการใช้งานตามนี้สำหรับแต่ละฟังก์ชัน API (endpoint) ต่อหนึ่งช่อง (channel) สำหรับข้อมูลเพิ่มเติมเกี่ยวกับขอบเขตของข้อจำกัดการใช้งาน ดูที่ Scope of rate limits

# API Rate Limits

This document outlines the rate limits for various API endpoints. Rate limits help ensure fair usage and protect the service from abuse. Each endpoint has specific rate limits as described below:

## Messaging Endpoints
| Endpoint                                      | Rate Limit                 |
|-----------------------------------------------|----------------------------|
| Send a narrowcast message                     | 60 requests per hour        |
| Send a broadcast message                      | 60 requests per hour        |
| Get number of message deliveries              | 60 requests per hour        |
| Get number of friends                         | 60 requests per hour        |
| Get friend demographics                       | 60 requests per hour        |
| Get user interaction statistics               | 60 requests per hour        |
| Get statistics per unit                       | 60 requests per hour        |
| Test webhook endpoint                         | 60 requests per hour        |

| Add user IDs or Identifiers for Advertisers (IFAs) to an audience (by JSON) | 60 requests per minute |
| Add user IDs or Identifiers for Advertisers (IFAs) to an audience (by file) | 60 requests per minute |

| Create message impression audience            | 60 requests per minute      |

| Get a list of shared audiences in Business Manager   | 60 requests per minute      |



## Webhook Endpoints
| Endpoint                                      | Rate Limit                 |
|-----------------------------------------------|----------------------------|

## Miscellaneous Endpoints
| Endpoint                                      | Rate Limit                 |

## Other API Endpoints

> \* Denotes rate limits for specific use cases and might have additional constraints.

[ข้อควรระวัง] อย่าส่งคำขอเกินข้อจำกัดการใช้งาน: หากคุณส่งคำขอที่เกินข้อจำกัดการใช้งาน คุณจะได้รับข้อความแสดงข้อผิดพลาดที่ระบุว่า 429 Too Many Requests. เมื่อลงมือพัฒนา LINE Bot โดยใช้ Messaging API ควรปฏิบัติตามแนวทางการพัฒนา Messaging API รวมถึงแนวทางเกี่ยวกับข้อจำกัดการใช้งาน


### Status Code
These HTTP status codes are returned after an API call. We follow the HTTP status code specification (opens new window)unless otherwise stated.
[Status Code](https://developers.line.biz/en/reference/messaging-api/#rate-limits)

### Response headers
The following HTTP headers are included in Messaging API responses

|Response headers	|Description|
| -                             | -                         |
X-Line-Request-Id	|Request ID. An ID is issued for each request.
X-Line-Accepted-Request-Id (Not always included)	|If the request has already been accepted using the same retry key, the x-line-request-id of the accepted request is shown. For more information, see Retrying an API request.

> **เคล็ดลับ:** เก็บค่า `X-Line-Request-Id` ไว้ใน log เสมอ เวลาติดต่อ LINE Support ขอให้ debug request ที่ fail เขาจะถาม id นี้เป็นสิ่งแรก

## รู้จักข้อความทุกประเภทใน Messaging API

หลังจากรู้แล้วว่า endpoint อยู่ที่ไหนและยิงได้บ่อยแค่ไหน ทีนี้มาดูว่า "พัสดุ" ที่เราจะส่งมีกี่แบบ — แต่ละแบบเหมาะกับ use case ที่ต่างกัน

### Text message (v2)
Ref: https://developers.line.biz/en/reference/messaging-api/#text-message-v2

Text message (v2) สามารถแทนที่ string ที่อยู่ภายใน `{` และ `}` ด้วย mention และ emoji ได้ ต่างจาก text message แบบเดิม สามารถใช้ text message แบบเดิมได้ต่อไป แต่ฟีเจอร์ใหม่ในอนาคตอาจเพิ่มเฉพาะใน text message (v2) เท่านั้น

```json
// Text message (v2) example with substitution
{
  "type": "textV2",
  "text": "Hello {user}! Welcome to {shop}",
  "substitution": {
    "user": {
      "type": "mention",
      "mentionee": {
        "type": "user",
        "userId": "U1234567890abcdef..."
      }
    },
    "shop": {
      "type": "emoji",
      "productId": "5ac1bfd5040ab15980c9b435",
      "emojiId": "001"
    }
  }
}
```

### Template message
Ref: https://developers.line.biz/en/reference/messaging-api/#template-messages

Template message เป็นข้อความที่มี layout สำเร็จรูป ช่วยให้สร้างประสบการณ์ที่สมบูรณ์ยิ่งขึ้นให้กับผู้ใช้ ใช้ [actions](https://developers.line.biz/en/docs/messaging-api/actions/) เพื่อให้ผู้ใช้โต้ตอบกับ bot ได้ง่ายเพียงแตะ ไม่ต้องพิมพ์ข้อความ

Template ที่ใช้ได้มีดังนี้:
- **Buttons** - มีพื้นที่สำหรับรูปภาพ, หัวข้อ, ข้อความ และปุ่ม action
- **Confirm** - มีพื้นที่สำหรับข้อความและปุ่ม 2 ปุ่ม
- **Carousel** - มีหลายคอลัมน์ที่ผู้ใช้สามารถเลื่อนดูได้
- **Image carousel** - มีหลายรูปภาพที่ผู้ใช้สามารถเลื่อนดูได้

หากต้องการส่งข้อความที่มี layout ยืดหยุ่นมากขึ้น ให้ใช้ [Flex Message](https://developers.line.biz/en/docs/messaging-api/message-types/#flex-messages) แทน

#### Buttons template
Ref: https://developers.line.biz/en/reference/messaging-api/#buttons

Buttons template มีพื้นที่สำหรับรูปภาพ, หัวข้อ, ข้อความ และปุ่ม action นอกจากปุ่มแล้ว ยังสามารถตั้ง action บนพื้นที่รูปภาพ, หัวข้อ หรือข้อความได้ด้วย

```json
// Buttons template message example
{
  "type": "template",
  "altText": "This is a buttons template",
  "template": {
    "type": "buttons",
    "thumbnailImageUrl": "https://example.com/bot/images/image.jpg",
    "imageAspectRatio": "rectangle",
    "imageSize": "cover",
    "imageBackgroundColor": "#FFFFFF",
    "title": "Menu",
    "text": "Please select",
    "defaultAction": {
      "type": "uri",
      "label": "View detail",
      "uri": "http://example.com/page/123"
    },
    "actions": [
      {
        "type": "postback",
        "label": "Buy",
        "data": "action=buy&itemid=123"
      },
      {
        "type": "postback",
        "label": "Add to cart",
        "data": "action=add&itemid=123"
      },
      {
        "type": "uri",
        "label": "View detail",
        "uri": "http://example.com/page/123"
      }
    ]
  }
}
```

## ข้อผิดพลาดที่มักเจอ

- **พลาด:** ยิง `POST https://api.line.me/v2/bot/message/{messageId}/content` แล้วได้ 404
  **ถูก:** endpoint นี้อยู่ที่ **api-data.line.me** ไม่ใช่ `api.line.me` — จำง่าย ๆ ว่า "ไฟล์ binary = data domain"

- **พลาด:** ตั้ง cron ยิง broadcast ทุก 30 วินาที แล้วโดน 429 หลังจากยิงได้ไม่นาน
  **ถูก:** broadcast/narrowcast จำกัด **60 ครั้งต่อชั่วโมง** (เฉลี่ย 1 ครั้ง/นาที) — ถ้าอยากยิงถี่กว่านั้นให้ใช้ push/multicast แทน

- **พลาด:** ส่ง text message ที่มีอักขระเกิน 5,000 แล้ว API ตีกลับ 400
  **ถูก:** ตัดข้อความเป็นหลาย ๆ object ภายใน `messages` array (สูงสุด 5 ข้อความต่อ request) หรือใช้ Flex message ที่รองรับ layout ซับซ้อนกว่า

- **พลาด:** ใช้ `type: "template"` แต่ลืมใส่ `altText` — API return error ว่า "May not be empty"
  **ถูก:** Template, Flex, Imagemap ทุกอันต้องมี `altText` เพราะใช้แสดงใน notification และในอุปกรณ์ที่ render ไม่ได้

- **พลาด:** ตั้งค่า `previewImageUrl` เป็นไฟล์ขนาด 5 MB แล้วข้อความไม่ส่ง
  **ถูก:** preview image ต้อง **≤ 1 MB** ส่วนรูปเต็ม (`originalContentUrl`) ≤ 10 MB — และต้องเป็น HTTPS (TLS 1.2+) เท่านั้น

- **พลาด:** ไม่เก็บ `X-Line-Request-Id` ใน log เวลาเจอปัญหาแล้วคุยกับ LINE Support ไม่ได้
  **ถูก:** log header นี้ทุก response เสมอ มันคือ "เลขตามรอย" ที่ฝั่ง LINE ใช้ค้น request ย้อนหลังได้

- **พลาด:** ใช้ HTTP (ไม่ใช่ HTTPS) สำหรับ `originalContentUrl` ของ image/video แล้วผู้ใช้เห็นรูปโหลดไม่ออก
  **ถูก:** LINE บังคับ HTTPS เท่านั้น ถ้าเซิร์ฟเวอร์ยังเป็น HTTP ให้ผ่าน CDN เช่น Cloudflare หรือ Firebase Hosting

## Checklist ก่อนไปต่อ

- [ ] รู้แล้วว่า endpoint ไหนอยู่ที่ `api.line.me` และอันไหนอยู่ที่ `api-data.line.me`
- [ ] เข้าใจว่า broadcast/narrowcast มีลิมิต 60/ชม. ส่วน reply/push ได้ 2,000/วินาที
- [ ] รู้จัก type ของ message ทั้งหมด: text, sticker, image, video, audio, location, imagemap, textV2, coupon, template, flex
- [ ] รู้ว่า text จำกัด 5,000 อักขระ และ 1 request ใส่ได้สูงสุด 5 messages
- [ ] log `X-Line-Request-Id` ทุกครั้งสำหรับ debug
- [ ] เตรียม URL รูป/วิดีโอเป็น HTTPS พร้อม TLS 1.2+
