# Workshop: Get Statistics of Sent Messages — วัดผลทุกข้อความที่ส่งไป

> ส่งโปรโมชันไปหาลูกค้า 500 คน — แต่จะรู้ได้ยังไงว่าคนเปิดอ่านจริงกี่คน? คลิกลิงก์ไหนบ้าง? ดูวิดีโอจบกี่เปอร์เซ็นต์? ถ้าไม่มีข้อมูลสถิติ การยิง LINE message ก็เหมือน "ส่งใบปลิวแบบหลับตา" — LINE Messaging API มี endpoint สำหรับดึงสถิติที่เรียกว่า **Message Event Aggregation** ที่ให้คุณวัดผลแคมเปญได้ละเอียด โดยยังคงปกป้องความเป็นส่วนตัวของผู้ใช้

## ทำไมต้องรู้เรื่องนี้?

สามารถดึงข้อมูลสถิติเกี่ยวกับการปฏิสัมพันธ์ของผู้ใช้กับข้อความแบบ push และ multicast ที่คุณส่งไปยังผู้ใช้หลายรายได้ โดยสามารถกำหนดหน่วยรวมข้อมูล (aggregation unit) เพื่อแยกสถิติออกเป็นกลุ่มๆ ตามที่คุณต้องการ

โดยปกติแล้ว คุณไม่สามารถดึงสถิติเกี่ยวกับ action ที่ผู้ใช้ทำกับข้อความ (เช่น การเปิดอ่านข้อความ หรือการแตะ URL) เพื่อปกป้องความเป็นส่วนตัวของผู้ใช้ แต่สามารถทำได้โดยการรวมข้อมูลเป็นหน่วย (unit) ที่คุณกำหนดเอง ทำให้ไม่สามารถระบุตัวตนของบุคคลใดบุคคลหนึ่งได้ — นี่คือ **privacy-preserving analytics** ที่ LINE ออกแบบมาให้

## ภาพรวม: Statistics Aggregation Flow

```mermaid
flowchart TD
    Send[ส่งข้อความ Push/Multicast<br/>พร้อม customAggregationUnit] --> Wait[รอ 2-3 วัน<br/>ให้ผู้ใช้มีเวลา engage]
    Wait --> Query[GET /v2/bot/insight/message/event/aggregation<br/>?customAggregationUnit=xxx]
    Query --> Check{ผู้ใช้ unique<br/>ที่ engage ≥ 20?}
    Check -->|น้อยกว่า 20| Null[แสดง null<br/>ปกป้องความเป็นส่วนตัว]
    Check -->|≥ 20 คน| Stats[แสดงสถิติจริง<br/>impression, click, mediaPlayed]
    Stats --> Analyze[วิเคราะห์ open rate,<br/>click rate, video completion]

    Narrow[Narrowcast/Broadcast] --> Send2[ได้ requestId]
    Send2 --> WaitN[รอ 2-3 วัน]
    WaitN --> Query2[GET /v2/bot/insight/message/event<br/>?requestId=xxx]
    Query2 --> Stats
```

## What Are Message Statistics?

หลังจากกำหนด unit name ขณะส่งข้อความแล้ว คุณสามารถดึงสถิติของข้อความในแต่ละหน่วยได้ โดยจะมีข้อมูลดังนี้:

- จำนวนผู้ใช้ที่เปิดอ่านข้อความ (unique impression)
- จำนวนผู้ใช้ที่เปิด URL ใดๆ ในข้อความ (unique click)
- จำนวนผู้ใช้ที่เริ่มเล่นวิดีโอหรือเสียงในข้อความ (unique media played)

การใช้สถิติเหล่านี้ช่วยให้คุณสามารถวิเคราะห์พฤติกรรมของผู้ใช้ และวัดผลประสิทธิภาพของการสื่อสารได้

### ตัวอย่างข้อมูลสถิติที่ได้

| จำนวนผู้รับ | จำนวนการเปิดข้อความ | อัตราการเปิด | จำนวนการคลิกลิงก์ | อัตราการคลิกลิงก์ |
|--------------|----------------------|----------------|---------------------|----------------------|
| 500          | 433                  | 87%            | 323                 | 65%                  |

## หมายเหตุเกี่ยวกับข้อมูลสถิติแบบรวม (Notes on Aggregated Statistics)

ข้อมูลสถิติที่ได้อาจมีความคลาดเคลื่อนเล็กน้อย เพื่อปกป้องความเป็นส่วนตัวของผู้ใช้ ระบบจะ **แสดงค่าเป็น null (ไม่มีข้อมูล)** ในกรณีดังต่อไปนี้:

1. ค่าสถิติรวม (aggregated statistics) น้อยกว่า 20
2. ถึงแม้ว่าค่าสถิติรวมจะมากกว่าหรือเท่ากับ 20 แต่ **จำนวนผู้ใช้จริงที่มีส่วนร่วมกับเหตุการณ์นั้นๆ มีน้อยกว่า 20 คน**

### ตัวอย่าง

หากมีข้อมูลดังนี้:
- จำนวนครั้งที่มีการเริ่มเล่นวิดีโอ = 30
- จำนวนผู้ใช้ที่เริ่มเล่นวิดีโอ = 15

ผลลัพธ์ที่ได้จะถูกแสดงเป็น:

```json
{
  "videoPlayCount": null,
  "uniqueUsersPlayedVideo": null
}
```

เพราะจำนวน unique users (15) น้อยกว่า threshold (20 คน) — นี่คือ mechanism เพื่อกันการ re-identify ผู้ใช้รายบุคคล

## การจัดการจำนวน Unit Name สูงสุดต่อเดือน

### ขีดจำกัดของ Unit Name ต่อเดือน

ภายในเดือนปัจจุบัน (ตั้งแต่วันที่ 1 ถึงวันสุดท้ายของเดือน) คุณสามารถส่งข้อความพร้อมกำหนดชื่อหน่วย (unit name) ได้สูงสุด **1,000 แบบ** ต่อเดือน

### ตัวอย่าง

- หากคุณส่งข้อความในเดือนมีนาคม โดยใช้ชื่อหน่วยตั้งแต่ `promotion_0001` ถึง `promotion_1000` (รวม 1,000 แบบ)
  - ในเดือนถัดไป (เมษายน) คุณสามารถ:
    - ใช้ชื่อเดิม (`promotion_0001` ถึง `promotion_1000`) ได้อีกครั้ง
    - หรือใช้ชื่อใหม่ (`promotion_1001` ถึง `promotion_2000`) ก็ได้

### ข้อควรระวัง

หากคุณส่งข้อความโดยใช้ชื่อหน่วยเกินขีดจำกัด เช่น `promotion_0001` ถึง `promotion_1500` (รวม 1,500 แบบ):

- เฉพาะชื่อ `promotion_0001` ถึง `promotion_1000` เท่านั้นที่จะถูกบันทึกเป็น unit name
- ตั้งแต่ `promotion_1001` เป็นต้นไป **จะไม่ถูกจัดเก็บเป็น unit name**
  - ข้อความจะถูกส่ง แต่ **ไม่สามารถดึงสถิติแบบแยกตาม unit name ได้**

> **หมายเหตุ:** คุณไม่สามารถกำหนดหรือเปลี่ยนแปลง unit name ของข้อความ push หรือ multicast ได้หลังจากที่ส่งข้อความไปแล้ว

### การตรวจสอบก่อนและหลังส่งข้อความ

หากคุณมี unit name จำนวนมาก ให้ตรวจสอบว่า unit name ถูกกำหนดได้หรือถูกกำหนดแล้วโดยใช้วิธีใดวิธีหนึ่งต่อไปนี้:

- **ก่อนส่งข้อความ** — ใช้ endpoint [Get the number of unit name types assigned during this month](https://developers.line.biz/en/reference/messaging-api/#get-the-number-of-unit-name-types-assigned-during-this-month) เพื่อตรวจสอบว่าจำนวน unit name ของเดือนปัจจุบันยังไม่ถึง 1,000
- **หลังส่งข้อความ** — ใช้ endpoint [Get a list of unit names assigned during this month](https://developers.line.biz/en/reference/messaging-api/#get-a-list-of-unit-names-assigned-during-this-month) เพื่อตรวจสอบว่า unit name ที่กำหนดไว้มีอยู่ในรายการ

## ตรวจสอบจำนวน Unit Name ที่ใช้งาน

คุณสามารถตรวจสอบจำนวนชื่อหน่วยที่กำหนดไว้ในเดือนปัจจุบันได้โดยใช้ API:

- [Get the number of unit name types assigned during this month](https://developers.line.biz/en/reference/messaging-api/#get-the-number-of-unit-name-types-assigned-during-this-month)

```
GET https://api.line.me/v2/bot/message/aggregation/info
```

```bash
curl -v -X GET https://api.line.me/v2/bot/message/aggregation/info \
-H 'Authorization: Bearer {channel access token}'
```

นอกจากนี้ คุณยังสามารถดึงรายชื่อ unit name ที่ถูกกำหนดในเดือนปัจจุบันได้โดยใช้ endpoint [Get a list of unit names assigned during this month](https://developers.line.biz/en/reference/messaging-api/#get-a-list-of-unit-names-assigned-during-this-month) ทั้งนี้ ไม่มี endpoint สำหรับตรวจสอบ unit name ที่ถูกกำหนดก่อนเดือนที่แล้ว

---

> แม้จะมีข้อจำกัดในการ "กำหนดชื่อหน่วยได้สูงสุด 1,000 แบบต่อเดือน" แต่หากคุณดึงข้อมูลสถิติย้อนหลังสำหรับช่วงเวลาที่มีข้อความส่งออกไป คุณจะยังสามารถดูข้อมูลของ **unit name ทั้งหมดที่อยู่ในช่วงเวลานั้น** ได้ (หากเคยถูกกำหนดในช่วงเวลาก่อนหน้า)

---

## Example: วัดผลข้อความจริงแบบ end-to-end

### 1. ส่งข้อความโดยกำหนด Unit Name

```bash
curl -v -X POST https://api.line.me/v2/bot/message/multicast \
-H 'Content-Type: application/json' \
-H 'Authorization: Bearer {channel access token}' \
-d '{
    "to": ["U4af4980629...","U0c229f96c4...",...], // 150 user IDs
    "messages":[
        {
            "type": "text",
            "text": "Our new product is available now!\nhttps://example.com/new-item/"
        }
    ],
    "customAggregationUnits": [
        "new-item-message-yyyymmdd"
    ]
}'
```

เคล็ดลับการตั้งชื่อ `customAggregationUnits`:
- ใส่วันที่ลงในชื่อ เช่น `promo-blackfriday-20260419` — จะได้แยกแคมเปญได้ง่าย
- ตั้งชื่อให้สั้น อ่านง่าย เข้าใจได้ (จะโชว์ใน dashboard)
- ห้ามซ้ำกับของเดือนอื่น ถ้าอยากวัดผลรายแคมเปญ

### 2. ดึงและรวมสถิติ

> **Timing Guidance:** ควรรอ **สองสามวัน** หลังจากส่งข้อความ เพื่อให้ข้อมูลสถิติมีความถูกต้องและครบถ้วนมากที่สุด เนื่องจากผู้ใช้อาจเปิดอ่านข้อความหรือคลิกลิงก์ในภายหลัง

ใช้ API เพื่อดึงสถิติตาม unit name ที่กำหนดไว้:

```bash
curl -v -X GET https://api.line.me/v2/bot/insight/message/event/aggregation \
-H 'Authorization: Bearer {channel access token}' \
--data-urlencode 'customAggregationUnit=new-item-message-yyyymmdd' \
--data-urlencode 'from=20210301' \
--data-urlencode 'to=20210331' \
-G
```

ตัวอย่างผลลัพธ์ที่ได้:

```json
{
  "overview": {
    "uniqueImpression": 111,
    "uniqueClick": 74,
    "uniqueMediaPlayed": null,
    "uniqueMediaPlayed100Percent": null
  },
  "messages": [
    {
      "seq": 1,
      "impression": 111,
      "uniqueImpression": 111,
      "mediaPlayed": null,
      "mediaPlayed25Percent": null,
      "mediaPlayed50Percent": null,
      "mediaPlayed75Percent": null,
      "mediaPlayed100Percent": null,
      "uniqueMediaPlayed": null,
      "uniqueMediaPlayed25Percent": null,
      "uniqueMediaPlayed50Percent": null,
      "uniqueMediaPlayed75Percent": null,
      "uniqueMediaPlayed100Percent": null
    }
  ],
  "clicks": [
    {
      "seq": 1,
      "url": "https://example.com/new-item/",
      "click": 74,
      "uniqueClick": 74,
      "uniqueClickOfRequest": 74
    }
  ]
}
```

จากข้อมูลสถิติข้างต้น สามารถนำมาวิเคราะห์อัตราการเปิดอ่านและอัตราการคลิกลิงก์ได้:

| จำนวนผู้รับ | จำนวนการเปิดข้อความ | อัตราการเปิด | จำนวนการคลิกลิงก์ | อัตราการคลิกลิงก์ |
|--------------|----------------------|----------------|---------------------|----------------------|
| 150          | 111                  | 74%            | 74                  | 67%                  |

สังเกตว่า `uniqueMediaPlayed` เป็น `null` ในตัวอย่างนี้ — เพราะข้อความไม่มี media หรือมี unique viewer น้อยกว่า 20 คน

## สถิติข้อความ Narrowcast และ Broadcast

นอกจากสถิติของข้อความแบบ push และ multicast แล้ว คุณยังสามารถดึงสถิติการปฏิสัมพันธ์ของผู้ใช้กับข้อความแบบ **narrowcast** หรือ **broadcast** ที่ส่งจาก LINE Official Account ของคุณได้

โดยใช้ endpoint **Get user interaction statistics**:

```
GET https://api.line.me/v2/bot/insight/message/event
```

```bash
curl -v -X GET https://api.line.me/v2/bot/insight/message/event \
-H 'Authorization: Bearer {channel access token}' \
--data-urlencode 'requestId={request_id}' \
-G
```

สำหรับรายละเอียดเพิ่มเติม ดูที่ [Get user interaction statistics](https://developers.line.biz/en/reference/messaging-api/#get-message-event) ใน Messaging API reference

> **หมายเหตุ:** สถิติของ narrowcast/broadcast จะดึงข้อมูลโดยใช้ `requestId` ที่ได้รับจากการส่งข้อความ ซึ่งแตกต่างจากสถิติแบบ unit-based ที่ใช้ `customAggregationUnit` ควรรอสองสามวันหลังส่งข้อความเพื่อให้ได้ข้อมูลสถิติที่ครบถ้วน

## เปรียบเทียบวิธีดึงสถิติ

| ประเภทข้อความ | วิธีดึงสถิติ | Parameter |
|-------------|------------|-----------|
| Push / Multicast | `GET /v2/bot/insight/message/event/aggregation` | `customAggregationUnit` |
| Narrowcast / Broadcast | `GET /v2/bot/insight/message/event` | `requestId` |

## ข้อผิดพลาดที่มักเจอ

- **พลาด:** ดึงสถิติทันทีหลังส่งข้อความ แล้วได้ข้อมูลไม่ครบ (บางค่าเป็น 0 หรือน้อยกว่าความเป็นจริง)
  **ถูก:** รอ 2-3 วันหลังส่ง แล้วค่อยดึง — เพราะผู้ใช้อาจเปิดข้อความในภายหลัง

- **พลาด:** ตั้ง `customAggregationUnits` เป็นชื่อเดียวกันในทุกแคมเปญ แล้วแยกผลไม่ออก
  **ถูก:** ใส่ identifier ไม่ซ้ำ เช่น `campaign-newyear-2026`, `promo-20260419`

- **พลาด:** ใช้ unit name เกิน 1,000 แบบในเดือนเดียว แล้ว unit ที่เกินไม่มีสถิติ
  **ถูก:** ใช้ `GET /v2/bot/message/aggregation/info` เช็คก่อนส่ง — ถ้าเต็มแล้วรวม unit หลายแคมเปญเข้าด้วยกัน

- **พลาด:** เจอ `null` แล้วคิดว่า API พัง
  **ถูก:** `null` เป็น privacy-preserving feature — มีผู้ engage น้อยกว่า 20 ต้องเพิ่มขนาด audience

- **พลาด:** พยายามเปลี่ยน unit name หลังส่งแล้ว
  **ถูก:** unit name แก้ไม่ได้หลังส่ง — ต้องตั้งชื่อให้ดีตั้งแต่แรก

- **พลาด:** ใช้ `customAggregationUnit` กับ broadcast/narrowcast แล้วไม่ได้ผล
  **ถูก:** broadcast/narrowcast ใช้ `requestId` ไม่ใช่ `customAggregationUnit`

## Checklist ก่อนไปต่อ

- [ ] เข้าใจ 3 metric หลัก: impression, click, mediaPlayed
- [ ] ใส่ `customAggregationUnits` ในทุกข้อความ push/multicast ที่อยากวัดผล
- [ ] เก็บ `requestId` ทุกครั้งที่ส่ง broadcast/narrowcast
- [ ] รอ 2-3 วันก่อนดึงสถิติ
- [ ] เข้าใจว่า `null` หมายถึง unique users < 20 (privacy protection)
- [ ] เช็คจำนวน unit name ที่ใช้ไปแล้วในเดือนนั้น (`/message/aggregation/info`)
- [ ] ออกแบบระบบให้เก็บสถิติลง DB เพื่อดู trend ข้ามเดือน
