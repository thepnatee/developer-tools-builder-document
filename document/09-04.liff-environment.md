## liff.getContext()

`liff.getContext()` ใช้สำหรับดึงข้อมูลบริบท (context) ของ LIFF app ที่กำลังทำงานอยู่ โดยจะคืนค่าเป็น object ที่มี property สำคัญดังนี้:

- **type**: ประเภทของหน้าจอที่เปิด LIFF app ได้แก่ `utou` (แชท 1-on-1), `group` (กลุ่ม), `room` (ห้องแชท), `external` (เบราว์เซอร์ภายนอก), `none` (หน้าจออื่น เช่น Wallet tab)
- **viewType**: ขนาดของ LIFF app view (`compact`, `tall`, `full`) จะคืนค่าเฉพาะเมื่อ type ไม่ใช่ `external`
- **userId**: User ID ของผู้ใช้
- **liffId**: LIFF ID ของแอป
- **endpointUrl**: URL ของ service endpoint
- **accessTokenHash**: ครึ่งแรกของค่า SHA256 hash ของ access token ใช้สำหรับตรวจสอบความถูกต้องของ access token
- **scope**: array ของ scope ที่ LIFF app มีสิทธิ์ใช้งาน เช่น `openid`, `email`, `profile`, `chat_message.write`
- **availability**: object ที่ระบุว่าฟีเจอร์ต่าง ๆ ของ LIFF พร้อมใช้งานหรือไม่ในสภาพแวดล้อมปัจจุบัน
- **menuColorSetting**: การตั้งค่าสีของ header ใน LIFF browser

```javascript
const context = liff.getContext();
console.log(context);
// ตัวอย่างผลลัพธ์:
// {
//   type: "utou",
//   userId: "U850014438e...",

//   viewType: "full",
//   endpointUrl: "https://example.com/",
//   scope: ["chat_message.write", "openid", "profile"],
//   ...
// }
```

## Action Button (ปุ่มแอคชัน)

LIFF app ที่ตั้งค่าขนาด view เป็น `Full` จะแสดงปุ่มแอคชัน (action button) ที่ header โดยอัตโนมัติ

เมื่อผู้ใช้กดปุ่มแอคชัน จะแสดงผลแตกต่างกันตามเวอร์ชันของแอป LINE:
- **LINE เวอร์ชัน 15.12.0 ขึ้นไป**: จะแสดง Multi-tab view
- **LINE เวอร์ชันต่ำกว่า 15.12.0**: จะแสดง Options (ตัวเลือก)

> **ซ่อนปุ่มแอคชัน**: สามารถเปิดใช้งาน **Module mode** ของ LIFF app ใน LINE Developers Console เพื่อซ่อนปุ่มแอคชันได้

## Multi-tab View

Multi-tab view จะแสดงตัวเลือกสำหรับ LIFF app ที่กำลังใช้งานอยู่ และบริการที่เพิ่งใช้งานล่าสุด ประกอบด้วย 2 ส่วน:

### 1. Options (ตัวเลือก)
| รายการ | คำอธิบาย |
| --- | --- |
| **Refresh** | โหลดหน้าปัจจุบันใหม่ |
| **Share** | แชร์ permanent link ของหน้าปัจจุบันผ่านข้อความ LINE |
| **Minimize browser** | ย่อ LIFF browser ให้เล็กลง |
| **Permission setting** | เปิดหน้าจอตั้งค่าสิทธิ์ (กล้อง, ไมโครโฟน) ใช้ได้ใน LINE เวอร์ชัน 14.6.0 ขึ้นไป |

### 2. Recently Used Services (บริการที่เพิ่งใช้งาน)
- แสดง LIFF app ที่ผู้ใช้เคยเปิด เรียงตามลำดับการใช้งานล่าสุด สูงสุด 50 รายการ
- เมื่อผู้ใช้ปิด LIFF app หรือเปิด LIFF app ใหม่ จะถ่ายภาพหน้าจอเก็บไว้เป็นประวัติการใช้งาน
- ผู้ใช้สามารถเปิด LIFF app อีกครั้งจากประวัติการใช้งานได้

**การ Resume และ Reload:**

| พฤติกรรม | เงื่อนไข | รายละเอียด |
| --- | --- | --- |
| **Resume** (กลับมาทำงานต่อ) | LIFF app ที่ใช้งานภายใน 12 ชั่วโมงล่าสุด และอยู่ใน 10 รายการล่าสุด | กลับมาที่หน้าจอเดิม, access token, ประวัติการเรียกดู และตำแหน่ง scroll จะถูกเก็บไว้ |
| **Reload** (โหลดใหม่) | ไม่เข้าเงื่อนไขของ Resume | LIFF app จะถูก initialize ใหม่ที่ URL ล่าสุด, access token, ประวัติการเรียกดู และตำแหน่ง scroll จะถูกยกเลิก |

**เงื่อนไขการแสดงใน Recently Used Services:**
- แอป LINE เวอร์ชัน 15.12.0 ขึ้นไป
- ขนาด LIFF app ต้องเป็น `Full`
- Module mode ของ LIFF app ต้องปิดอยู่

> **หมายเหตุ**: `liff.sendMessages()` ไม่สามารถใช้งานได้หลังจาก LIFF app ถูก reload จาก recently used services หากต้องการใช้ `liff.sendMessages()` หลัง reload ให้เปิด LIFF app ใหม่โดยแตะ LIFF URL ในห้องแชท

## Module Mode

Module mode เป็นการตั้งค่าใน LINE Developers Console สำหรับ LIFF app โดยมีผลดังนี้:
- **เมื่อเปิด Module mode**: ปุ่มแอคชัน (action button) ใน header จะถูกซ่อน และ LIFF app จะไม่แสดงในส่วน Recently Used Services ของ Multi-tab view
- **เมื่อปิด Module mode** (ค่าเริ่มต้น): ปุ่มแอคชันจะแสดงตามปกติ และ LIFF app จะปรากฏใน Recently Used Services

## LIFF Browser Cache (การจัดการแคชของ LIFF Browser)

LIFF browser ใช้ [WKWebView](https://developer.apple.com/documentation/webkit/wkwebview) บน iOS และ [Android WebView](https://developer.android.com/reference/android/webkit/WebView) บน Android ซึ่งอาจบันทึกและใช้เนื้อหาที่แสดงเป็นแคชตามคำสั่งของ HTTP headers เช่น [Cache-Control](https://developer.mozilla.org/ja/docs/Web/HTTP/Reference/Headers/Cache-Control)

**วิธีควบคุมแคช:**
- ใช้ HTTP headers เช่น `Cache-Control` เพื่อควบคุมพฤติกรรมการแคชใน LIFF browser
- ตัวอย่าง: ตั้ง `Cache-Control: no-cache` หรือ `Cache-Control: no-store` เพื่อป้องกันการแคช

> **หมายเหตุ**: ไม่มีวิธีลบแคชที่เก็บไว้ใน LIFF browser โดยตรง ดังนั้นการตั้งค่า Cache-Control header อย่างเหมาะสมตั้งแต่เริ่มต้นจึงเป็นสิ่งสำคัญ
