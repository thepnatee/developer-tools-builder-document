# Narrowcast


<p align="center" width="100%">
    <img  src="../assets/Narrowcast.webp
    "> 
</p>
เป็นฟีเจอร์หนึ่งของ LINE Messaging API ที่ช่วยให้ผู้พัฒนาและเจ้าของธุรกิจสามารถส่งข้อความไปยังกลุ่มผู้ใช้ที่เฉพาะเจาะจงได้ แทนที่จะส่งไปหาผู้ใช้ทั้งหมดเหมือนการใช้ Broadcast ข้อความ โดย Narrowcast จะช่วยให้ส่งข้อความไปยังผู้ใช้ตามเกณฑ์ที่กำหนด เช่น ข้อมูลประชากร ความสนใจ หรือพฤติกรรมการใช้งาน

Narrowcast ไม่สามารถส่งข้อความไปยังกลุ่มแชทหรือแชทหลายคนได้ ใช้ได้เฉพาะการส่งถึงผู้ใช้รายบุคคลเท่านั้น

ข้อดีของ Narrowcast:

- ช่วยให้การส่งข้อความตรงกลุ่มเป้าหมายมากขึ้น
- ประหยัดค่าใช้จ่ายในการส่งข้อความ
- สามารถทดสอบข้อความก่อนส่งแบบเต็มรูปแบบได้ (A/B Testing)
- การใช้ Narrowcast จะมีเงื่อนไขและขอบเขตที่ผู้พัฒนาอาจต้องทำตาม เช่น การขออนุญาตจากผู้ใช้ให้ส่งข้อมูลไปยังกลุ่มที่มีเงื่อนไขที่เจาะจง


---

## ขั้นตอนการส่ง Narrowcast Message

การส่ง Narrowcast message มี 3 ขั้นตอนหลัก:

1. **เตรียม Audience หรือ Request ID** - สร้างกลุ่มเป้าหมายที่ต้องการส่งข้อความ
2. **เริ่มส่ง Narrowcast message** - กำหนด recipient, demographic filter, และ limit object
3. **ตรวจสอบสถานะการส่ง** - ตรวจสอบว่าข้อความถูกส่งสำเร็จหรือไม่

---

## ประเภทของ Audience ที่สร้างได้ผ่าน Messaging API

| ประเภท Audience | คำอธิบาย |
|----------------|----------|
| Audience for uploading user IDs | กลุ่มผู้ใช้ที่ระบุด้วย user ID หรือ IFA (Identifier For Advertisers) |
| Message click audience | กลุ่มผู้ใช้ที่คลิก URL ในข้อความที่ส่งไป |
| Message impression audience | กลุ่มผู้ใช้ที่เปิดอ่านข้อความที่ส่งไป |

### ประเภท Audience ที่ไม่สามารถสร้างผ่าน Messaging API ได้

ประเภทต่อไปนี้ต้องสร้างผ่าน LINE Official Account Manager หรือ LINE Ad Manager:

- Chat tag audience
- Friend path audience
- Reservation audience
- Rich menu impression audience
- Rich menu click audience
- Web traffic audience (LINE Tag)
- Web traffic audience (Tracking Tag)
- App event audience
- Video view audience
- Image click audience
- LINE Beacon Network ad impression audience

## การเตรียมกลุ่มเป้าหมาย (Prepare Audience)

ตารางด้านล่างแสดงวิธีเตรียม audience ตามประเภทผู้รับที่ต้องการ สามารถรวมเงื่อนไขหลายอย่างได้โดยใช้ logical operators (AND, OR, NOT):

| กลุ่มเป้าหมาย | ข้อมูลที่ต้องเตรียม |
|---------------|-------------------|
| ผู้ใช้ทุกคนที่เป็นเพื่อนกับ LINE Official Account | ไม่ต้องเตรียมข้อมูลเพิ่มเติม |
| ผู้ใช้ที่ระบุด้วย user ID หรือ IFA | สร้าง Audience for uploading user IDs (แบบ JSON หรือ File) |
| ผู้ใช้ที่คลิก URL ในข้อความ | สร้าง Message click audience |
| ผู้ใช้ที่เปิดอ่านข้อความ | สร้าง Message impression audience |
| ผู้ใช้ที่ได้รับ narrowcast message ก่อนหน้า | ระบุ request ID ของ narrowcast ที่ส่งก่อนหน้าใน redelivery object |
| ผู้ใช้ที่มี tag เฉพาะในแชท | Chat tag audience - สร้างผ่าน LINE Official Account Manager |
| ผู้ใช้ที่เพิ่มเพื่อนผ่าน path เฉพาะ | Friend path audience - สร้างผ่าน LINE Official Account Manager |

## Retargeting Audiences

Retargeting audience คือกลุ่มเป้าหมายที่สร้างจากพฤติกรรมของผู้ใช้ต่อข้อความที่ส่งไปก่อนหน้า มี 2 ประเภท:

### Message Click Audience

สร้างกลุ่มเป้าหมายจากผู้ใช้ที่คลิก URL ในข้อความที่ส่งไป:

```Shell
curl -v -X POST https://api.line.me/v2/bot/audienceGroup/click \
-H 'Authorization: Bearer {channel access token}' \
-H 'Content-Type: application/json' \
-d '{
    "description": "audienceGroupName_click",
    "requestId": "request_id_of_sent_message",
    "clickUrl": "https://example.com/path"
}'
```

- `requestId` - request ID ที่ได้รับจากการส่งข้อความ (ต้องสร้างภายใน 60 วันหลังส่งข้อความ)
- `clickUrl` - URL ที่ต้องการติดตามการคลิก (ไม่บังคับ หากไม่ระบุจะนับการคลิก URL ทั้งหมดในข้อความ)

### Message Impression Audience

สร้างกลุ่มเป้าหมายจากผู้ใช้ที่เปิดอ่านข้อความที่ส่งไป:

```Shell
curl -v -X POST https://api.line.me/v2/bot/audienceGroup/imp \
-H 'Authorization: Bearer {channel access token}' \
-H 'Content-Type: application/json' \
-d '{
    "description": "audienceGroupName_impression",
    "requestId": "request_id_of_sent_message"
}'
```

- `requestId` - request ID ที่ได้รับจากการส่งข้อความ (ต้องสร้างภายใน 60 วันหลังส่งข้อความ)

> **สำคัญ**: Retargeting audience (ทั้ง click และ impression) ต้องมีผู้ใช้อย่างน้อย 50 คนจึงจะใช้งานได้

## ตรวจสอบสถานะ Audience ก่อนส่งข้อความ

Audience ถูกสร้างแบบ asynchronous ดังนั้นก่อนส่ง narrowcast message ต้องตรวจสอบว่าสถานะของ audience เป็น `READY` แล้ว:

```Shell
curl -v -X GET https://api.line.me/v2/bot/audienceGroup/{audienceGroupId} \
-H 'Authorization: Bearer {channel access token}'
```

หาก `status` ใน response เป็น `READY` แสดงว่า audience พร้อมใช้งานสำหรับการส่งข้อความ

## 👥 ข้อกำหนดของกลุ่มเป้าหมาย (Audience Specification)

รายการข้อกำหนดสำหรับการใช้งานกลุ่มเป้าหมาย (Audience) ในระบบ Messaging API มีดังนี้:

| รายการ                         | ข้อกำหนด |
|-------------------------------|-----------|
| จำนวนกลุ่มเป้าหมายต่อช่องทาง (Channel) | สูงสุด 1,000 กลุ่ม |
| ระยะเวลาเก็บรักษาข้อมูล (Retention period) | สูงสุด 180 วัน (15,552,000 วินาที) |
| จำนวน user ID หรือ IFA ที่อัปโหลดได้ต่อคำขอเพื่อสร้าง audience | <ul><li>**JSON**: สูงสุด 10,000 รายการ</li><li>**File**: สูงสุด 1,500,000 รายการ</li></ul> |
| จำนวนผู้ใช้ต่อกลุ่มเป้าหมาย | <ul><li>**Audience จากการอัปโหลด user ID**: ไม่จำกัด</li><li>**Audience จากการคลิกข้อความ (message click)**: อย่างน้อย 50 คน</li><li>**Audience จากการเห็นข้อความ (message impression)**: อย่างน้อย 50 คน</li></ul> |
| ระยะเวลาสูงสุดในการสร้างกลุ่มเป้าหมายสำหรับการทำ Retargeting หลังจากส่งข้อความ | ภายใน 60 วัน (5,184,000 วินาที) |

---

> 🔎 **หมายเหตุเพิ่มเติม**  
- Audience ที่สร้างจากการคลิกหรือเห็นข้อความจะต้องมีอย่างน้อย 50 ผู้ใช้จึงจะสามารถใช้งานได้  
- การสร้างกลุ่มเป้าหมาย Retargeting จะต้องดำเนินการภายใน 60 วันหลังจากส่งข้อความนั้น

## Recipient Object (ออบเจกต์ผู้รับ)

เมื่อส่ง narrowcast message ให้ระบุเนื้อหาข้อความใน `messages` property และกลุ่มเป้าหมายใน `recipient` property หากไม่ระบุ `recipient` ข้อความจะส่งถึงผู้ใช้ทุกคนที่เป็นเพื่อนกับ LINE Official Account

สามารถระบุได้ 2 ประเภท:

### Audience Object

ใช้สำหรับส่งข้อความไปยัง audience ที่สร้างไว้:

```json
{
  "type": "audience",
  "audienceGroupId": 5614991017776
}
```

### Redelivery Object

ใช้สำหรับส่งข้อความไปยังผู้ใช้ที่ได้รับ narrowcast message ก่อนหน้า โดยระบุ request ID (`X-Line-Request-Id`) ที่ได้รับจากการส่ง narrowcast ครั้งก่อน:

```json
{
  "type": "redelivery",
  "requestId": "5b59509c-c57b-11e9-aa8c-2a2ae2dbcce4"
}
```

> **หมายเหตุ**: หากส่งข้อความด้วย redelivery object แล้วได้ `errorCode` เป็น `2` (จำนวนผู้รับไม่เพียงพอ) อาจเกิดจาก:
> - ผู้รับบางส่วนได้บล็อก LINE Official Account หลังจากได้รับข้อความ narrowcast ครั้งก่อน
> - จำนวนผู้รับลดลงเนื่องจากการใช้ operators (AND หรือ NOT) ร่วมกับ audience หรือ demographic filter อื่น

---

## Demographic Filter Object (ตัวกรองข้อมูลประชากร)

ใช้กำหนดเงื่อนไขตามคุณลักษณะของผู้ใช้ผ่าน `filter.demographic` property ได้แก่ เพศ, อายุ, ประเภท OS, พื้นที่, และระยะเวลาการเป็นเพื่อน

ตัวอย่างการกรองตามเพศ:

```json
{
  "type": "gender",
  "oneOf": ["male", "female"]
}
```

ตัวอย่างการกรองตามอายุ:

```json
{
  "type": "age",
  "gte": "age_20",
  "lt": "age_25"
}
```

ตัวอย่างการกรองตามประเภทแอป:

```json
{
  "type": "appType",
  "oneOf": ["android", "ios"]
}
```

ตัวอย่างการกรองตามพื้นที่:

```json
{
  "type": "area",
  "oneOf": ["jp_23", "jp_05"]
}
```

ตัวอย่างการกรองตามระยะเวลาการเป็นเพื่อน:

```json
{
  "type": "subscriptionPeriod",
  "gte": "day_7",
  "lt": "day_30"
}
```

## Operator Object (ตัวดำเนินการ)

ใช้รวมเงื่อนไขหลายอย่างเข้าด้วยกันโดยใช้ตัวดำเนินการเชิงตรรกะ:

| Operator | คำอธิบาย | ใช้กับ |
|----------|----------|--------|
| **AND** | เซตร่วม (product set) - ผู้ใช้ต้องตรงเงื่อนไขทุกข้อ | recipient object, demographic filter |
| **OR** | เซตรวม (union set) - ผู้ใช้ตรงเงื่อนไขข้อใดข้อหนึ่ง | recipient object, demographic filter |
| **NOT** | เซตผลต่าง (difference set) - ไม่รวมผู้ใช้ที่ตรงเงื่อนไข | recipient object, demographic filter |

### ตัวอย่างการใช้ AND operator กับ recipient:

```json
"recipient": {
    "type": "operator",
    "and": [
        {
            "type": "audience",
            "audienceGroupId": 5614991017776
        },
        {
            "type": "audience",
            "audienceGroupId": 5614991017777
        }
    ]
}
```

### ตัวอย่างการใช้ NOT operator กับ recipient:

```json
"recipient": {
    "type": "operator",
    "and": [
        {
            "type": "audience",
            "audienceGroupId": 5614991017776
        },
        {
            "type": "operator",
            "not": {
                "type": "redelivery",
                "requestId": "5b59509c-c57b-11e9-aa8c-2a2ae2dbcce4"
            }
        }
    ]
}
```

### ตัวอย่างการใช้ OR operator กับ demographic filter:

```json
"filter": {
    "demographic": {
        "type": "operator",
        "or": [
            {
                "type": "gender",
                "oneOf": ["male"]
            },
            {
                "type": "age",
                "gte": "age_20",
                "lt": "age_30"
            }
        ]
    }
}
```

### การใช้ Operator แบบซ้อน (Nested Structure)

สามารถซ้อน operator objects เพื่อสร้างเงื่อนไขที่ซับซ้อนได้ โดย operator ที่อยู่ลึกที่สุดจะถูกประมวลผลก่อน

ตัวอย่าง: ต้องการส่งถึง "ผู้ใช้ที่อยู่ใน AudienceA AND AudienceB AND NOT (AudienceC AND AudienceD) AND AudienceE":

```json
{
    "type": "operator",
    "and": [
        {
            "type": "audience",
            "audienceGroupId": "AudienceA"
        },
        {
            "type": "audience",
            "audienceGroupId": "AudienceB"
        },
        {
            "type": "operator",
            "not": {
                "type": "operator",
                "and": [
                    {
                        "type": "audience",
                        "audienceGroupId": "AudienceC"
                    },
                    {
                        "type": "audience",
                        "audienceGroupId": "AudienceD"
                    }
                ]
            }
        },
        {
            "type": "audience",
            "audienceGroupId": "AudienceE"
        }
    ]
}
```

## Limit Object (ออบเจกต์จำกัดจำนวน)

ใช้กำหนดจำนวนสูงสุดของข้อความ narrowcast ที่จะส่ง หากมีการจำกัดจำนวน ระบบจะเลือกผู้รับแบบสุ่ม

```json
{
  "max": 100,
  "upToRemainingQuota": true,
  "forbidPartialDelivery": true
}
```

| Property | คำอธิบาย |
|----------|----------|
| `max` | จำนวนสูงสุดของข้อความที่จะส่ง (ไม่บังคับ) |
| `upToRemainingQuota` | หาก `true` จะจำกัดการส่งไม่เกินโควตาข้อความรายเดือนที่เหลือ (ค่าเริ่มต้น: `false`) |
| `forbidPartialDelivery` | หาก `true` จะยกเลิกการส่งทั้งหมดหากไม่สามารถส่งถึงผู้รับทุกคนได้ (ค่าเริ่มต้น: `false`) |

### ตัวอย่างการควบคุมจำนวนการส่งด้วย Limit Object

| เงื่อนไข | Target reach: 100, Monthly limit: 90, Target recipients: 80 | Target reach: 100, Monthly limit: 50, Target recipients: 80 |
|----------|------|------|
| ไม่ระบุ limit object | ❌ Request error (Target reach เกิน monthly limit) | ❌ Request error (Target reach เกิน monthly limit) |
| ไม่ระบุ `max`, `upToRemainingQuota`=`true`, `forbidPartialDelivery`=`false` | ✅ ส่งถึงผู้รับทั้งหมด | ✅ ส่ง 50 คน (ภายใน monthly limit) |
| ไม่ระบุ `max`, `upToRemainingQuota`=`true`, `forbidPartialDelivery`=`true` | ✅ ส่งถึงผู้รับทั้งหมด | ❌ ยกเลิกเนื่องจาก partial delivery |
| `max`=30, `upToRemainingQuota`=`true`, `forbidPartialDelivery`=`false` | ✅ ส่ง 30 คน (เท่ากับ max) | ✅ ส่ง 30 คน (เท่ากับ max) |
| `max`=30, `upToRemainingQuota`=`true`, `forbidPartialDelivery`=`true` | ❌ ยกเลิกเนื่องจาก partial delivery | ❌ ยกเลิกเนื่องจาก partial delivery |

**คำอธิบายเงื่อนไข:**
- **Target reach** - จำนวนผู้ใช้ที่สามารถเข้าถึงได้ทั้งหมด
- **Monthly limit** - ขีดจำกัดประมาณการของการส่งข้อความในเดือนปัจจุบัน
- **Target recipients** - ผู้รับที่กรองด้วย attributes หรือ audiences แล้ว

## ข้อจำกัดโควตาข้อความรายเดือน (Monthly Message Quota)

จำนวนข้อความที่ส่งได้มีขีดจำกัดรายเดือน หากพยายามส่งเกินขีดจำกัด การส่งจะล้มเหลว

- ตั้งค่า `limit.upToRemainingQuota` เป็น `true` เพื่อให้ระบบจำกัดการส่งไม่เกินโควตาที่เหลืออัตโนมัติ
- จำนวนข้อความนับจากจำนวนผู้รับ ไม่ใช่จำนวน message objects ในคำขอ

> **คำเตือน**: เมื่อส่ง narrowcast message ระบบอาจจอง quota ไว้ตามจำนวนผู้รับทั้งหมดที่ประมาณไว้ แม้ว่าจำนวนจริงที่ส่งอาจน้อยกว่า ทำให้ข้อความอื่นอาจส่งไม่ได้จนกว่าการส่ง narrowcast จะเสร็จสมบูรณ์ หากพยายามส่งข้อความในสถานะนี้ จะได้ error `You have reached your monthly limit.`

## จำนวนผู้รับขั้นต่ำ (Minimum Recipient Requirements)

เพื่อป้องกันไม่ให้สามารถคาดเดาคุณลักษณะของผู้รับได้ ระบบจะไม่อนุญาตให้ส่ง narrowcast message หากจำนวนผู้รับน้อยกว่าจำนวนขั้นต่ำที่กำหนด

> **Narrowcast messages can't be sent if the number of recipients is less than the required amount**

ดูรายละเอียดเพิ่มเติมที่: [Restrictions on sending messages using attributes and audiences](https://developers.line.biz/en/reference/messaging-api/#send-narrowcast-message-restrictions)

### ความสัมพันธ์ระหว่างการตั้งค่า `max`, `upToRemainingQuota` และ `forbidPartialDelivery` กับจำนวนการจองและการส่งสูงสุด

| `max` ตั้งไว้หรือไม่ | `upToRemainingQuota` ตั้งไว้หรือไม่ | จำนวนการจองและการส่งสูงสุด |
|----------------------|-----------------------------|------------------------------|
| ไม่ได้ตั้งค่า (`Not set`) | `false`                     | จำนวนเป้าหมายที่เข้าถึงได้ทั้งหมด |
| ตั้งค่าเป็นตัวเลขใดๆ       | `false`                     | ค่าต่ำสุดระหว่างจำนวนเป้าหมายที่เข้าถึงได้และค่าที่ตั้งใน `max` |
| ไม่ได้ตั้งค่า (`Not set`) | `true`                      | ค่าต่ำสุดระหว่างจำนวนเป้าหมายที่เข้าถึงได้และขีดจำกัดประมาณการของเดือนปัจจุบัน |
| ตั้งค่าเป็นตัวเลขใดๆ       | `true`                      | ค่าต่ำสุดระหว่างจำนวนเป้าหมายที่เข้าถึงได้, ขีดจำกัดประมาณการของเดือนปัจจุบัน และค่าที่ตั้งใน `max` |

> **`forbidPartialDelivery`**: หากตั้งเป็น `true` และไม่สามารถส่งถึงผู้รับทุกคนได้ (เช่น เกินโควตา) ระบบจะยกเลิกการส่งทั้งหมดแทนที่จะส่งเพียงบางส่วน
